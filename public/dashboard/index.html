Technical API Documentation

Gunakan panduan ini untuk mengintegrasikan pengiriman pesan Blast dan Screening Nomor melalui HTTP request ke server Anda.

1. Authentication

Header: Authorization: Bearer <jwt_token>

2. Manajemen Sesi Karyawan

POST /api/sessions

Membuat sesi Playwright headless baru untuk Karyawan yang login. Mengembalikan session_id yang digunakan di semua API lainnya.

{ "success": true, "session_id": "812b32da-...", "message": "Sesi berhasil dibuat, silakan scan QR Code", "status": "pending_qr" }

DELETE /api/sessions/:id

Hentikan dan hapus sesi aktif beserta browser Playwright di *backend*. Sangat disarankan dipanggil jika perangkat *disconnect* / saat agen menekan tombol Log Out.

3. Blast Message Endpoint

POST /api/rcs/send

  • Body JSON:
  • recipients (Array): Kumpulan nomor HP, misal ["081234", "089876"]
  • message (String): Isi teks promosi atau notifikasi.
  • session_id (String, optional): ID spesifik device pengirim, biarkan kosong agar sistem mencarinya secara otomatis.
{ "recipients": ["081234567890"], "message": "Halo, ini pesan blast promosi via sistem." }

Contoh Response (Success):

{ "success": true, "message": "1 pesan berhasil masuk antrean.", "message_ids": [45], "session_id": "7475ba65-a709-4a0f-be61-620acad2f9b8" }

4. Screening Nomor (Cek RCS)

POST /api/rcs/screen

  • Body JSON:
  • phone_numbers (Array): Kumpulan nomor HP yang mau discreening.
{ "phone_numbers": ["08111111111", "08222222222"] }

Contoh Response (Success):

{ "success": true, "message": "2 nomor dimasukkan ke antrean screening." }

GET /api/rcs/screen/results

Mendapatkan hasil antrean screening. Parameter kembalian is_rcs_capable: 1 berarti Support RCS. 0 berarti jatuh ke SMS.

{ "success": true, "data": [ { "phone_number": "08111111111", "is_rcs_capable": 1, "status": "done" } ] }

5. Alur Integrasi QR Code (Untuk Frontend Karyawan)

Panduan integrasi alur kerja menyambungkan sesi Google Messages pada layar kerja (dashboard Frontend) agen atau karyawan Anda.

  • 1. Meminta Sesi Baru
    Panggil POST /api/sessions dengan Token JWT karyawan untuk mendapatkan respon session_id.
  • 2. Visualisasi QR Code (& Auto Refresh)
    Gunakan tag elemen gambar standar yang diarahkan ke file Stream API. Frontend wajib me-refresh gambar menggunakan tambahan stempel waktu (timestamp) agar gambar QR direfresh secara manual oleh browser setiap 2 detik.
<!-- HTML --> <img id="qr-karyawan" src="http://[SERVER_IP]:3000/api/sessions/[SESSION_ID]/qr/image" style="background: white; padding: 10px;" /> <!-- Javascript --> <script> const qrImage = document.getElementById('qr-karyawan'); const sessionId = 'AMBIL_DARI_LANGKAH_1'; // Refresh gambar QR setiap 2 detik tanpa me-refresh halaman (mencegah kedip) const intervalGambar = setInterval(() => { qrImage.src = `http://[SERVER_IP]:3000/api/sessions/${sessionId}/qr/image?t=${new Date().getTime()}`; }, 2000); </script>
  • 3. Cek Status Penyambungan
    Selama gambar diperbarui, panggil GET /api/sessions/[SESSION_ID]/status setiap 3 detik. Jika hasilnya merespon "active", hapus interval gambar via Javascript dan sembunyikan kotak QR Code-nya.
